3 LangGraph agent · planner → executor → synthesizer

plan ▸ run three tools ▸ synthesize

explicit allowlist
The planner is a deterministic ranker over data/destinations.csv — no model hop for mechanical work. It picks the destination, the counterfactual, and the feature profile the ML tool will classify. Only three tool names are accepted; anything else is refused.
backend/app/agent/graph.py backend/app/agent/registry.py · ALLOWED_TOOL_NAMES structured tool errors → LLM can reason about them
why planner-then-executor? Tool order is fixed (RAG → ML → live) and the tools don't depend on each other. ReAct's reflective loops are wasted spend on a coarse, deterministic flow. Defended in docs/PLANNER_VS_REACT.md.
